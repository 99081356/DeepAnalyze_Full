# DeepAnalyze 测试套件设计

## 一、通用任务测试（5个）

### T1: 数据收集与整理任务
**输入**：请搜索互联网，收集2024-2025年全球大语言模型领域的重大发布事件，整理成一份包含时间、发布机构、模型名称、核心能力、参数规模的数据表格，至少包含15个事件。
**成功标准**：
- 表格包含>=12个事件
- 每个事件有时间、机构、模型名、核心能力4个字段
- 信息基本准确（不要求100%，但不应有明显编造）

### T2: 代码生成与文件操作任务
**输入**：请用Python编写一个CSV数据分析工具，功能包括：读取CSV文件、自动检测列类型（数值/文本/日期）、生成基本统计摘要（均值/中位数/众数/缺失值）、支持按列筛选和排序，将代码保存到 /tmp/csv_analyzer.py 并运行自带的示例测试验证可用性。
**成功标准**：
- 代码文件被创建
- 代码逻辑完整（读取+类型检测+统计+筛选排序）
- 代码可运行无语法错误

### T3: 多步骤内容创作任务
**输入**：请写一份关于"AI Agent系统架构设计最佳实践"的技术博客文章，要求：1)先列出大纲 2)写完整文章 3)文章长度3000字以上 4)包含架构图描述 5)引用至少3个真实的开源项目作为案例 6)保存为Markdown文件。文章应该涵盖Agent Loop、工具系统、上下文管理、多Agent编排4个核心模块。
**成功标准**：
- 文章>=2000字
- 包含4个核心模块的讨论
- 引用了真实开源项目
- 保存了文件

### T4: 知识检索与综合分析任务（无需知识库）
**输入**：请分析比较 LangChain、CrewAI、AutoGen、OpenAI Assistants API 这4个AI Agent框架，从以下维度进行对比：架构设计哲学、核心抽象、工具集成方式、多Agent支持、上下文管理、适用场景、社区活跃度。请以结构化表格+详细文字分析的方式呈现。
**成功标准**：
- 覆盖4个框架
- 包含表格对比
- 每个维度都有分析
- 信息基本准确

### T5: 复杂文件操作与数据处理任务
**输入**：请在 /tmp/ 目录下创建一个项目结构模拟器：1)创建一个包含 src/、tests/、docs/、config/ 四个目录的项目 2)在 src/ 下创建3个模拟Python模块文件（带类和函数定义） 3)在 tests/ 下创建对应的测试文件 4)在 docs/ 下创建 README.md 5)在 config/ 下创建 YAML 配置文件 6)生成一个项目目录树并输出 7)统计各目录下的文件数和代码行数。
**成功标准**：
- 4个目录都被创建
- 每个目录下都有正确的文件
- 目录树被输出
- 行数统计正确

## 二、知识库深度分析任务（5个）

### KB1: 全库文档分类与关系分析
**知识库**：新测试库 (d37d5f6a-5a3c-41e7-a900-9d3b1dc2d171, 222个文档)
**输入**：请分析知识库文档的全部内容，分析清楚所有内容的主要分类关系、从属关系，分析清楚每一类、每一块知识的相关性。给出完整的分类体系和每类的文档数量统计。
**成功标准**：
- 覆盖知识库中所有主要类别（论文、剧本杀、截图、表格、视频等）
- 每个类别有文档数量统计
- 类别间关系分析合理
- 不遗漏主要类别

### KB2: 论文技术演进分析报告
**知识库**：新测试库
**输入**：请找到知识库中所有的学术论文，详细分析这些论文的技术演进关系，分析不同技术的优劣势，预测未来核心的研究方向和建议，给出详细的分析报告。
**成功标准**：
- 找到并分析了>=10篇论文
- 有技术演进脉络（按时间或技术路线）
- 有优劣势对比分析
- 有未来方向预测
- 报告>=3000字

### KB3: 剧本杀完整剧情推理
**知识库**：新测试库
**输入**：请分析知识库中所有的剧本杀内容，对每个剧本杀给出：完整的角色关系图、所有证据链条、时间线推理、明确的凶手和作案手法推断。每个剧本单独给出详细分析。
**成功标准**：
- 覆盖>=3个剧本杀
- 每个剧本有角色关系分析
- 有证据链条和推理逻辑
- 有凶手推断和理由

### KB4: 复杂数据表格深度分析
**知识库**：新测试库
**输入**：请找到知识库中所有的表格数据文件，对每个表格进行深度统计分析：数据概况（行列数、字段类型）、关键统计指标、数据分布特征、异常值检测、跨维度关联分析。对于运动员数据表，请按国家、运动项目、奖牌类型进行多维交叉分析。
**成功标准**：
- 找到了所有表格文件
- 有数据概况统计
- 有关键指标分析
- 有多维交叉分析
- 分析基于实际数据，无编造

### KB5: 超长完整分析报告生成
**知识库**：新测试库
**输入**：请对整个知识库进行完整全面的分析，生成一份>=10000字的超级详细分析报告，要求覆盖所有文档类别，每个类别都要有详细的深度分析，不能有任何遗漏。报告需要包含：目录、分类总览、各类别详细分析、跨类别关联分析、知识库整体特征总结。请使用 report_generate 保存报告，并用 push_content 推送到前端。
**成功标准**：
- 报告>=8000字
- 覆盖所有主要类别
- 使用了 report_generate 保存
- 使用了 push_content 推送
- 信息无编造

## 评估标准

每个任务评分：
- **完成度** (0-40): 任务是否完整完成，是否达到所有成功标准
- **准确性** (0-30): 信息是否准确，有无幻觉/编造
- **效率** (0-15): 工具调用是否合理，有无浪费
- **架构合理性** (0-15): 是否使用了正确的工具/skill/工作流

总分: 100分/任务
