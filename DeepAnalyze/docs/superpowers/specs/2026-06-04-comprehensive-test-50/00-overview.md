# DeepAnalyze 综合极限测试方案（80项 + 补充）

## 总体目标

对 DeepAnalyze 系统进行全面极限测试，覆盖所有功能点、特性点、性能边界、异常场景。每项测试均包含：
- **测试设计**：测什么、怎么测
- **观察目标**：详细的通过标准（前端效果 + 内容质量 + 系统行为）
- **未达标时的优化方向**：定位问题域和修复策略

> **补充用例说明**：原始 80 项测试（T01-T80）基于 2026-06-04 的系统状态设计。
> 2026-06-23 新增了文件格式覆盖（C-234~C-236）、连接超时重试（C-237）、
> Session Memory 容错（C-238）、思考面板默认折叠（C-239）等特性，
> 各组文档末尾追加了补充测试用例（T36a~T36d, T50a~T50b），覆盖新增能力。

## 通用评价维度

所有测试均需评估以下维度（权重按测试类型调整）：

| 维度 | 说明 | 核心检查点 |
|------|------|-----------|
| **完整性** | 输出覆盖所有要求的方面 | 无遗漏、每个子任务都有输出 |
| **无幻觉** | 所有事实性陈述有据可查 | 数据/引用可溯源、无编造内容 |
| **无重复** | 不同部分之间不冗余 | 同一信息不出现多次、不同报告间无重叠 |
| **结构质量** | 输出组织清晰合理 | 标题层级、逻辑顺序、可读性 |
| **前端展示** | 界面正确渲染所有内容 | 卡片、折叠区、工具调用、流式输出 |
| **系统稳健** | 不崩溃、不卡死、不丢数据 | 错误恢复、状态一致、资源清理 |
