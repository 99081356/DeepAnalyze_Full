# DeepAnalyze 迭代优化测试要求

> 本文档为迭代优化过程中的关键唤醒要求，上下文压缩后重读此文件恢复上下文。

---

## 核心原则

以 Agent 优化原则设计文档 `docs/superpowers/specs/agent-optimization-design.md` 为优化核心原则和目标，持续实现各层级优化→测试验证→再优化的迭代循环。

## 测试流程

### 用例级测试
1. 一个用例一个用例进行测试
2. 严格记录：测试用例、答复过程、工具调用过程、多Agent工作过程、推理路径、最终答案
3. 基于过程数据和答案数据，系统性思考最优优化方案
4. 每一组测试完成后综合分析，进行一轮优化
5. 答错的题放入待测试队列最末端，迭代直到全部答对

### 前端验证
- 使用 Playwright 前端界面操作，发出指令和观察输出
- 跟踪后端所有动作及输出
- 评估所有前端输出：文字流 + push_content 卡片 + 工具调用结果
- 像人类使用一样依次点击所有功能选项验证完整性

## 测试集

| 测试集 | 聚焦点 | 优化方向 |
|--------|--------|---------|
| Agent Long Bench | 长文本写作 | 长文本写作能力 |
| FActScore | 事实准确性 | 零幻觉、多源验证 |
| LongBench-Write | 长文本写作 | 写作深度和完整性 |
| GAIA L3 | Agent 综合能力 | 工具使用、推理、信息获取 |

按 10 个一组进行测试。

## 模型配置

- 主模型：MiniMax M3
- 辅助模型：GLM 5.1

## 优化布局原则

| 层级 | 适合放置的内容 |
|------|--------------|
| L1 工具层 | 工具能力增强、bug 修复 |
| L2 引导层 | 通用能力描述、局限性描述（禁止强制性用语） |
| L3 上下文层 | 压缩管线、上下文管理 |
| L4 Agent循环 | 轮次管理、输出恢复、卡住检测 |
| L5 编排层 | 多Agent协调、并行策略 |
| L6 场景层 | 专属 Skill/Plugin（如公检法场景） |

## 严格禁止项

1. **禁止减损 Agent 通用能力**：子Agent maxTurns=200、主Agent无限制，不允许修改减小
2. **禁止固定超时时间**：根据观测情况灵活调整
3. **禁止场景特定提示词**：所有提示词必须考虑通用性
4. **禁止强制性用语**："必须"、"至少N个"等不能出现在 L2 引导中
5. **禁止为某项测试做定制化优化**：先反问别的场景是否同样适用

## 输出完整性

- 所有输出内容（文字流 + push_content）视为完整结果
- 所有输出和 push 内容必须持久化到前端同一内容中
- 模型输出长度设置为对应最大值（0 = 最大值）
- 输出不完整时后台发送"继续"让模型继续输出

## GAP 对标

持续对照优化原则设计文档，减小与 Claude Code 的能力差距，所有 GAP 都要无缩水、无简化地实现和补齐。
