## 优化措施（三轮迭代）

### 第一轮：L1（工具层）修复

| 修复 | 文件 | 层级 | 影响范围 |
|------|------|------|----------|
| **MiniMax 搜索 API 端点修复** | `tool-setup.ts`, `mcp-builtin-adapters.ts` | L1 | 从 `/v1/web_search` 更正为 `/v1/coding_plan/search`，请求体从 `{ query }` 修正为 `{ q }`，响应格式从 `data.results` 修正为 `organic` |
| **LLM 流式超时机制** | `openai-compatible.ts` | L1 | 新增连接超时 60s + 流式空闲超时 60s，防止模型无限挂起 |

### 第二轮：L4（Agent 循环）修复

| 修复 | 文件 | 层级 | 影响范围 |
|------|------|------|----------|
| **流式超时不重试** | `agent-runner.ts` `isTransientError()` | L4 | 流式空闲超时不再视为瞬时错误，直接触发模型切换 |
| **减少主模型重试次数** | `agent-runner.ts` `chatStreamWithFallback()` | L4 | 从 3 次降至 1 次，加快向备选模型切换 |
| **备选模型自动发现** | `router.ts` `getAlternateChatModel()` | L4 | 当配置的 fallback 与主模型相同时，自动搜索其他 chat 模型 |

### 第三轮：全面覆盖备用模型路径

| 修复 | 文件 | 层级 | 影响范围 |
|------|------|------|----------|
| **chatWithFallback 备选路径** | `agent-runner.ts` | L4 | 非流式 chat 路径也使用 getAlternateChatModel |
| **空响应回退备选路径** | `agent-runner.ts` | L4 | 空响应触发模型切换时也使用 getAlternateChatModel |
| **默认输出 token 预算提升** | `types.ts` | L4 | outputTokenBudget 从 16384 提升到 32768 |
| **finish 工具输出捕获** | `run-4benchmark-hard.ts` | 测试框架 | 捕获 finish 工具输出作为内容，解决 Agent 摘要在 finish 中的情况 |
| **超时时保留已积累数据** | `run-4benchmark-hard.ts` | 测试框架 | AbortError 时保留已接收的 content 和 toolCalls |
| **工具测试超时从 180s 提升到 300s** | `run-4benchmark-hard.ts` | 测试框架 | 为模型切换预留更多时间（主模型超时 60s + 备选模型执行 240s） |

### 修复效果总结

| 修复 | 层级 | 影响范围 | 效果 |
|------|------|----------|------|
| MiniMax API 修正 | L1 | 所有搜索类工具 | "fetch failed" → HTTP 200 正常返回 |
| LLM 流式超时 60s | L1 | 所有 LLM 调用 | 防止模型无限挂起 |
| 超时不重试 + 减少重试 | L4 | 所有超时场景 | 3×60s=180s → 1×60s=60s，快速切换 |
| **备选模型自动发现** | L4 | 所有模型挂起场景 | Tool-02/06 从失败→成功 |
| **全路径覆盖** | L4 | 所有模型切换路径 | chatWithFallback + 空响应 + chatStream 三路径统一 |
| outputTokenBudget 提升 | L4 | 所有长文本生成 | 16384 → 32768 |

### 模型切换机制（新增能力）

| 组件 | 方法 | 作用 |
|------|------|------|
| `ModelRouter.getAlternateChatModel()` | 遍历所有 provider | 找到与主模型不同的 chat 模型 |
| `chatStreamWithFallback()` | 流式路径 | 主模型失败后自动切换备选 |
| `chatWithFallback()` | 非流式路径 | 同上（第三轮修复） |
| 空响应回退 | Agent 循环 | 连续空响应时切换模型 |

当前模型切换链：`minimax-text` (MiniMax-M2.7) → `main` (glm-5.1)
